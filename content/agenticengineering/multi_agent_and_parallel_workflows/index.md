## Resources
- [Claude Code: Agent Teams](https://code.claude.com/docs/en/agent-teams) - Official documentation on running multi-agent teams with Claude Code.
- [Claude Code: Best practices for agentic coding](https://www.anthropic.com/engineering/claude-code-best-practices) - Includes guidance on running headless agents, parallelizing work, and using sub-agents. ([HN discussion, 614 points](https://news.ycombinator.com/item?id=43735550))
- [Don't Build Multi-Agents](https://cognition.ai/blog/dont-build-multi-agents) - Cognition (Devin) argues against naive multi-agent architectures. Specialized sub-agents sharing context outperform fully independent agents passing messages.
- [FleetCode: Open-source UI for running multiple coding agents](https://github.com/built-by-as/FleetCode) - The HN discussion includes detailed practitioner experiences with parallel agent execution using git worktrees. ([HN discussion, 103 points](https://news.ycombinator.com/item?id=45518861))
- [Software Engineering Is Back](https://blog.alaindichiappari.dev/p/software-engineering-is-back) - How coding agents changed an entire development workflow. The 594-comment HN thread is full of multi-agent patterns and real-world experiences. ([HN discussion, 375 points](https://news.ycombinator.com/item?id=43735550))
